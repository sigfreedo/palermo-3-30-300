Obiettivo dello studio

Valutare l'accessibilità degli spazi verdi urbani a Palermo applicando il framework 3-30-300 (Konijnendijk, 2023) attraverso dataset aperti e metodologie replicabili in altri contesti urbani.

Nota metodologica: L'ordine di analisi inverte il framework originale, partendo dalla regola dei 300 metri (accessibilità) come indicatore prioritario di equità urbana.

1. MAPPATURA SPAZI VERDI

1.1 Dataset OpenStreetMap

Tag selezionati:

Categoria A - Parchi e giardini:

  • leisure=park - Parchi pubblici
  • leisure=garden - Giardini pubblici

Categoria B - Aree ricreative:

  • leisure=playground - Aree gioco bambini
  • leisure=dog_park - Aree cani
  • leisure=fitness_station - Aree fitness outdoor
  • landuse=recreation_ground - Aree ricreative generiche
  • landuse=village_green - Piazze verdi

Criteri di esclusione:

  • Accesso: access=private, access=residents, access=no
  • Tipologia: garden:type=roof_garden, operator=condominium
  • Altre categorie escluse: impianti sportivi, foreste, riserve naturali, verde agricolo, cimiteri, campi da golf

Query Overpass Turbo:

[out:json][timeout:60];
{{geocodeArea:Palermo}}->.searchArea;

// CATEGORY A - Parks & Gardens

  way["landuse"="recreation_ground"]["access"!="private"](area.searchArea);

1.2 Dataset Copernicus Urban Atlas 2021

Fonte: FUA IT005L3 (Palermo)

Categoria selezionata:

  • 14110 - Green urban areas (Public access)

Categorie escluse:

  • 14130 - Green urban areas (Unknown access conditions)
  • Altre categorie: verde privato, impianti sportivi, foreste, vegetazione erbacea

Processing:

  1. Download: FlatGeobuf da Copernicus Land Monitoring Service
  2. Conversione: FlatGeobuf → GeoJSON (via NeatoGeo)
  3. Filtraggio dimensionale: area ≥ 2.000 m² (184 → 181 spazi)
  4. Filtraggio geografico: solo territorio comunale (181 → 129 spazi)

Risultati:

  • Dataset finale: 129 spazi
  • Superficie totale: 3,36 km²

3. ACCESSIBILITÀ SPAZI VERDI (REGOLA DEI 300 METRI)

3.1 Copertura cittadina

Metodo: Area-Weighted Interpolation

Algoritmo:

import geopandas as gpd
from shapely.ops import unary_union

# 1. Buffer 300m attorno agli spazi verdi
buffers = spazi_verdi.buffer(300)
area_copertura = unary_union(buffers)

# 2. Calcolo per sezione
popolazione_coperta = 0
for idx, sezione in sezioni.iterrows():
    intersezione = sezione.geometry.intersection(area_copertura)
    frazione_coperta = intersezione.area / sezione.geometry.area
    pop_coperta = sezione['popolazione'] * frazione_coperta
    popolazione_coperta += pop_coperta

# 3. Percentuale
percentuale = (popolazione_coperta / popolazione_totale) * 100
Assunzioni:
  • Popolazione uniformemente distribuita nelle sezioni
  • Distanza euclidea (linea d'aria)

Risultati:

Dataset Copertura Popolazione coperta Non coperta
OpenStreetMap 38,76% 246.293 abitanti 389.146 abitanti
Copernicus 40,06% 254.533 abitanti 380.906 abitanti

3.2 Copertura per quartiere

Stesso algoritmo applicato alle 25 unità territoriali (quartieri).

3.3 Valutazione puntuale

Input utente:

  • Geolocalizzazione (lat/lon)
  • Indirizzo testuale → Google Maps Geocoding API

Calcolo distanza:

from shapely.geometry import Point

punto_utente = Point(lon, lat)
distanza_minima = min([
    punto_utente.distance(spazio.boundary) 
    for spazio in spazi_verdi
])

Output:

  • ≤ 300m, 1 spazio: nome, area, distanza
  • ≤ 300m, >1 spazi: numero + dettagli spazio più grande
  • > 300m: segnalazione + spazio più vicino

4. COPERTURA ARBOREA (REGOLA DEL 30%)

Fonte: Copernicus Tree Cover Density 2023
Risoluzione: 10m raster
Range: 0-100% per pixel

Calcolo città/quartiere:

import rasterio
import numpy as np

with rasterio.open('tcd_2023.tif') as src:
    data = src.read(1)
    mask = rasterio.features.geometry_mask(
        [area_interesse], 
        transform=src.transform, 
        invert=True, 
        out_shape=data.shape
    )
    copertura_media = np.mean(data[mask])

Metriche derivate:

deficit_pct = 30 - copertura_media
area_da_coprire = area_quartiere * (deficit_pct / 100)
alberi_necessari = area_da_coprire / 35  # chioma 35 m²

Risultato: Palermo 12,1% (nessun quartiere ≥ 30%)

5. ALBERI VISIBILI (REGOLA DEL 3)

Approccio: Crowdsourcing

Stack:

  • Backend: Cloudflare Worker (serverless)
  • Storage: Cloudflare KV
  • API REST:
    • POST /api/tree-count → Input conteggio (0-99)
    • GET /api/tree-stats → Statistiche aggregate

Metriche:

{
  total_responses: n,
  average_trees: sum(counts) / n,
  pct_meeting_rule: (count >= 3) / n * 100
}

8. LIMITAZIONI

Dataset:

  • OSM: qualità variabile (mapping volontario)
  • Copernicus: risoluzione 10m può sottostimare alberi isolati

Calcoli:

  • Distanza euclidea (ignora rete stradale e barriere)
  • Popolazione uniforme per sezione
  • Alberi visibili: campione non probabilistico

9. REPLICABILITÀ

Per applicare la metodologia ad altra città:

  1. Sostituire {{geocodeArea:Palermo}} in query Overpass
  2. Scaricare FUA Copernicus corrispondente
  3. Adattare codice ISTAT PRO_COM
  4. Definire unità territoriali (quartieri)
  5. Eseguire script Python con librerie indicate
